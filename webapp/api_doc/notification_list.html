获得通知列表接口
  • app上那个红色圆点该如何显示?
    • 1、当打开app的时候异步读取该接口,如果本地不存在则保存到本地(比如SQLite),并且设置 is_read == false (可以在本地多创建一个字段叫is_read)
    • 2、处理完第一步后,查找本地是否有未读的数据,如果有,则显示小红点
    • 3、在进入列表页面的时候直接显示本地通知即可.
    • 4、在点击详情的时候,打开WebView预览详情之前,通过ID修改本地为is_read==true.
    注意:在读接口,存本地这些操作都加上异常捕获,不要因为网络,或无数据,使得整个app挂掉
  • 接口地址

    /api/notification_list.htm

  • 请求参数
    名称 类型 是否必须 默认值 说明
    token string 否 注意,这个token不是必须的
    timestamp long 否 毫秒时间
  • 响应结果
    名称 类型 一定存在 示例 说明
    id string 是
    gmtCreate long 是 1470020552000 毫秒时间戳
    title string 是 标题
    public boolean 是 是否公开
  • 示例
    [
    {
        "content": "这是详情",
        "createUid": "",
        "gmtCreate": 1470020552000,
        "gmtWrite": null,
        "id": "XXX",
        "ownerId": "", 
        "title": "这是标题",
        "userId": "XXX",
        "writeUid": ""
    }
